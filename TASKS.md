## Учёт времени работы приложения AppMonitor

Добавить функциональность учёта времени работы самого приложения AppMonitor (uptime):

- **Что нужно сделать:**
  - Записывать в БД время запуска и остановки приложения
  - Хранить историю сессий работы (start_time, end_time, длительность)
  - Показывать статистику в UI: сколько времени приложение работало за день/неделю/месяц
  - Учитывать только активные сессии (когда мониторинг реально работал, а не когда приложение свёрнуто в трей)

- **Возможная реализация:**
  - Новая таблица `app_sessions` в `core/database.py`
  - Запись старта при инициализации `ActivityMonitor`
  - Запись остановки при graceful shutdown (сигнал закрытия окна)
  - Виджет статистики в `ui/` или расширение существующего `stats_dialog.py`

## Уведомление при выключенных уведомлениях для приложения

Придумать механизм, который уведомляет пользователя, если для приложения AppMonitor отключены системные уведомления Windows.

- **Что нужно сделать:**
  - Определить, включены ли уведомления для AppMonitor в настройках Windows (Notifications & Actions)
  - Если уведомления выключены — показать предупреждение при запуске или в трее
  - Возможно, использовать Windows API (через `pywin32`) для проверки состояния уведомлений для конкретного приложения

- **Варианты реализации (нужно исследовать):**
  1. Проверка через реестр Windows: `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Notifications\Settings\AppMonitor`
  2. Использовать `ctypes` / `pywin32` для вызова Windows Shell API
  3. Альтернатива: показывать toast-уведомление при запуске и проверять, доходит ли оно (если нет — значит уведомления выключены)

- **Дополнительно:**
  - Добавить пункт в настройки приложения с предупреждением
  - Показывать иконку в трее с индикатором статуса уведомлений
